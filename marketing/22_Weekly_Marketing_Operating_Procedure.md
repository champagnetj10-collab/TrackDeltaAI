# Weekly Marketing Operating Procedure

The operating rhythm that runs this whole system week to week. Where the [90-day calendar](02_90_Day_Content_Calendar.md) defines *what* content goes out, this defines *when planning, production, and review actually happen* — the standing operating procedure (SOP) for whoever runs marketing (founder, solo, in this system's current scale).

---

## 1. The Weekly Cycle

```
SUNDAY          MON–SAT              FRIDAY              SUNDAY (next)
Plan the week → Execute + engage → Weekly Ship Log post → Review & repeat
```

### Sunday — Planning Block (60-90 min)
- [ ] Review the current week's slot in the 13-week arc (file 02) — confirm the theme and long-form video
- [ ] Confirm which SF-## ideas (from file 03 or file 11) are slotted for the week, pulling toward whichever pillar needs weight per the last review (see §3)
- [ ] Confirm all video content for the week is either already edited or has a real production slot booked before Monday — never enter a week planning to write and edit same-day for every post
- [ ] Check the analytics dashboard (file 21) for last week's numbers, particularly Section B (content performance)
- [ ] Note anything real happening in the business this week that could feed Building in Public content (file 08) — a ship, a bug, a metric, a piece of feedback

### Monday–Saturday — Execution
- [ ] Publish per the day's slot in the calendar
- [ ] Daily: 15-30 min genuine engagement pass (replies, comments on others' content) — do not skip this in favor of only posting; per file 12, reply engagement materially affects distribution on TikTok/X specifically, and it's simply how trust gets built on Reddit/Discord/forums
- [ ] As real building-in-public material comes up organically (a bug, a decision, a piece of feedback), capture it immediately (a note, a screenshot) even if it won't be posted until Friday's ship log — these things are forgotten by end of week otherwise

### Friday — Ship Log + Weekly Roundup
- [ ] Post the Weekly Ship Log (file 08, template 1) on X and LinkedIn
- [ ] Post the Facebook weekly roundup (per file 02's cadence table)
- [ ] Quick gut-check: did this week's content stay honest to the brand voice guide? Any claim made that needs a correction posted?

### Sunday — Review (folds into next Sunday's planning block)
- [ ] Which pillar/platform actually drove activations this week, not just views (file 21 Section A/C)? Adjust next week's pillar weighting slightly if a clear signal exists — don't overreact to a single week's noise, but do notice a real multi-week trend
- [ ] Any creator/community outreach (file 13) due for follow-up this week?
- [ ] Any Discord (file 14) presence ritual owed this week?
- [ ] Log anything learned into a running internal notes doc (not part of this repo — a personal/team working doc) so patterns compound over time instead of being re-discovered

## 2. Monthly Cycle (overlays on the weekly one)

## 3. Decision Rules (so weekly planning doesn't require re-litigating strategy every time)

| Situation | Rule |
|---|---|
| A pillar (education/product-proof/building-in-public/community/culture) underperforms for 3+ consecutive weeks | Reduce its weight next month, don't abandon it after one bad week |
| A platform generates zero real engagement for a full month | Flag at the monthly review for a harder look — possibly deprioritize, but don't quit after week 1 (file 07 §"Success Criteria" already establishes this patience threshold) |
| Real news breaks (ship, bug, milestone, user story) | It preempts a generic scheduled post — the calendar is a default, not a constraint on real, better content |
| No real building-in-public material exists for the week | Post an evergreen education piece instead of forcing a hollow BIP post — never fabricate progress to fill the slot (file 08's closing rule) |
| A piece of content underperforms right after publishing | Leave it — per file 06 §5, only reconsider titles/thumbnails in the first 48 hours, and don't delete underperforming posts |

## 5. The One-Person-Team Reality Check

This SOP assumes a very small team, likely a single founder running marketing alongside product/engineering work. When time is scarce in a given week, protect in this priority order:
1. Daily engagement/reply pass (cheapest, highest trust-per-minute activity)
2. The Friday ship log (maintains the building-in-public thread — the single highest-leverage recurring content type for this specific audience)
3. Scheduled short-form content
4. Long-form video (most expensive to produce — acceptable to occasionally skip a week rather than ship a rushed, lower-quality video that undercuts the channel's consistency)

Never sacrifice #1 for #3/#4 — a week with no new video but real engagement beats a week with new videos and zero replies.
